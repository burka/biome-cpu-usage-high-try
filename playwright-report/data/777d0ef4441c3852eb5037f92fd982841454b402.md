  - link "Section titled “Glob syntax reference”":

  - paragraph: "Glob patterns are used to match paths of files and folders. Biome supports the following syntax in globs:"
  - list:
    - listitem:
      - code: "*"
      - text: matches zero or more characters. It cannot match the path separator
      - code: /
      - text: .
    - listitem:
      - code: "**"
      - text: recursively matches directories and files. This sequence must be used as an entire path component, so both
      - code: "**a"
      - text: and
      - code: b**
      - text: are invalid and will result in an error. A sequence of more than two consecutive
      - code: "*"
      - text: characters is also invalid.
    - listitem:
      - code: "[...]"
      - text: matches any character inside the brackets. Ranges of characters can also be specified, as ordered by Unicode, so e.g.
      - code: "[0-9]"
      - text: specifies any character between 0 and 9 inclusive.
    - listitem:
      - code: "[!...]"
      - text: is the negation of
      - code: "[...]"
      - text: ", i.e. it matches any characters"
      - strong: not
      - text: in the brackets.
    - listitem:
      - text: If the entire glob starts with
      - code: "!"
      - text: ", it is a so-called negated pattern. This glob only matches if the path"
      - emphasis: doesn’t
      - text: match the glob. Negated patterns cannot be used alone, they can only be used as
      - emphasis: exception
      - text: to a regular glob.
